# Repo Notes

Maintenance gotchas for agents working on **this** repo. Personal coding rules live in the global `engineering-rules` skill, not here.

- `skills/` contains hand-written repo-owned skill packages. Edit `skills/<name>/SKILL.md` for real local skills; these files may use `{{REPO_ROOT}}`.
- `rules/` and `knowledge/` contain document sources. Document-backed skills such as rule or knowledge wrappers are declared in `meta.ts`, not hand-written under `skills/`.
- `generated/` contains rendered or vendored installable skill bundles. `pnpm skills link` renders `localSkillSources` into it; `pnpm skills sync` copies vendored skills into it. Don't hand-edit generated bundles.
- `meta.ts` is the link config: `localSkillSources` (directory or document sources rendered into `generated/`), `installableSkills` (generated bundles linked into agent skill dirs), `alwaysOnInstructionSkills` (document-backed skills delivered to rule dirs for agents that need always-on instructions), and `linkTargets` (the per-destination/config table the linker iterates). Skill names come from the `Skill` const object (not a TS `enum` — Node runs these `.ts` in type-stripping mode, which forbids non-erasable syntax). Type definitions live in `scripts/lib/metaTypes.ts`.
- Scripts live in `scripts/`: `skills.ts` is the CLI dispatcher (a command registry), each command is a file under `commands/`, and shared plumbing lives in `lib/` — git/submodule, vendor sync, fs helpers, note indexing, validation, plus the linking primitives (`symlink.ts`), skill linking (`skillLinks.ts`), and rule linking (`ruleLinks.ts`). `commands/link.ts` is a thin orchestrator over `linkTargets`. Tests live under `scripts/tests/`; run `npm test`. Lint with `npm run lint`.
- `pnpm skills <command>` is the single entry point (`status`, `link`, `unlink`, `sync`, `init`, `check`, `cleanup`, `validate`, `note`). `pnpm skills link` renders configured local sources, then walks `linkTargets`: full `installableSkills` into `~/.codex/skills` and `~/.agents/skills`, normal skills into `~/.claude/skills` and `~/.config/opencode/skills`, `alwaysOnInstructionSkills` as markdown into `~/.claude/rules` and `~/.config/opencode/rules`, and the opencode `instructions` glob into `~/.config/opencode/opencode.json`.
